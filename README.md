# Маленькое путешествие домой

Интерактивная открытка‑путешествие по местам (карта + задания) — подарок на 8 марта.  
Проходишь места по очереди, выполняешь активности и в конце открывается поздравление.

## Технологии

- **React + TypeScript**
- **Vite**
- **CSS** (нежная весенняя палитра, мягкие карточки/кнопки)
- **pigeon-maps** (карта и маркеры)

## Как запустить

```bash
npm install
npm run dev
```

Сборка:

```bash
npm run build
```

## Структура проекта

Главное — все «настраиваемые данные» лежат в `src/data/`.

```
public/
  images/
    places/          # фото для карточек мест
    puzzles/         # фото для пазлов (3×3)
src/
  components/
    activities/      # все активности (квиз/пазл/морзе/признание/сердечки)
    map/             # карта и маркеры
    modals/          # модальные окна
    screens/         # экраны (загрузка, код, посадочный талон, карта)
    ui/              # небольшие UI-компоненты (письма, тизеры)
  constants/         # константы (например центр карты)
  data/              # данные: код доступа, места, тексты, активности
  styles/            # CSS (base, variables, компоненты)
  types/             # общие типы приложения
  utils/             # утилиты (например темы маркеров)
```

## Где что менять (для людей без опыта)

### Код доступа (экран с 4 цифрами)

Файл: `src/data/codeData.ts`

- **Что менять**: `ACCESS_CODE`

### Места на карте (10 мест, координаты, тексты, активности)

Файл: `src/data/places.ts`

У каждого места есть:

- **id**: номер места (уникальный)
- **name / subtitle**: заголовки
- **coordinates**: координаты маркера \([широта, долгота]\)
- **image**: картинка карточки места (путь из `public/`)
- **memoryText**: текст воспоминания
- **activity**: активность (квиз / пазл / морзе / признание / сердечки)

#### Как двигать маркер (переместить место на карте)

В `coordinates` меняешь две цифры:

- первая — **широта**
- вторая — **долгота**

Чтобы быстро получить координаты: открой Google Maps / Яндекс Карты, нажми на точку → скопируй координаты и вставь сюда.

### Фото для мест

Папка: `public/images/places/`  
В `src/data/places.ts` у места меняешь поле `image`, например:

- `image: '/images/places/my-photo.jpg'`

## Активности (как они устроены и как настраивать)

Все активности настраиваются в `src/data/places.ts` внутри поля `activity`.

### QuizActivity (викторина)

Тип: `type: 'quiz'`

- **questions**: массив вопросов
- в варианте ответа **правильный** отмечается `correct: true`
- после правильного ответа показывается `successText`
- переход на следующий вопрос **вручную** кнопкой **“Следующий вопрос ➔”**
- на последнем вопросе кнопка **“Всё вспомнила ✨”**

### PuzzleActivity (пазл 3×3 из фото)

Тип: `type: 'puzzle'`

- **textBefore / textAfter**: тексты до/после
- **imageUrl**: путь к картинке для пазла (берётся из `public/`)

#### Как подготовить фото для пазла

- **Лучше всего**: квадратное фото **900×900 px**
- **Куда положить**: `public/images/puzzles/`
- **Как указать** в `src/data/places.ts`:

`imageUrl: '/images/puzzles/place-name.jpg'`

Пазл режется на 9 частей автоматически (через CSS `background-position`), drag‑and‑drop сохраняется.

### MorseActivity (азбука Морзе)

Тип: `type: 'morse'`

- **encoded**: строка с точками/тире
- **answer**: правильный ответ (то, что должен ввести человек)
- **successText**: текст после успеха

В активности есть таблица‑легенда, чтобы можно было расшифровать прямо в игре.

### ConfessionActivity (кодовый замок из смайликов)

Тип: `type: 'confession'`

- показывается 3 ячейки с эмодзи
- по клику эмодзи переключаются
- правильная комбинация сейчас: **🧸 🧸 🧸**
- после правильного кода появляется `revealedText` + анимация успеха

Доступные эмодзи: 🧸 🌸 💌 ❤️ ⭐

### HeartsActivity (собирание сердечек)

Тип: `type: 'hearts'`

- **total**: сколько сердечек нужно собрать
- после заполнения — показывается `textAfter`

## Настройка карты

### Центр карты

Файл: `src/constants/mapConstants.ts` → `MOSCOW_CENTER`

### Цвета/вид маркеров (розовый/золотой)

Файл: `src/utils/helpers.ts`

- `PINK_MARKER` — для не посещённых
- `GOLD_MARKER` — для посещённых

## Публикация на GitHub Pages

В проекте уже есть скрипт деплоя через `gh-pages`.

### 1) Укажи правильный `base` для Vite

Файл: `vite.config.ts`

- если репозиторий называется, например, `little-trip`, то:
  - `base: '/little-trip/'`
- если это корневой сайт вида `username.github.io`, можно оставить:
  - `base: '/'`

### 2) Задеплой

```bash
npm run deploy
```

Скрипт соберёт проект и загрузит папку `dist` в ветку `gh-pages`.
